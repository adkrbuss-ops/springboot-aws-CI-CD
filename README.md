# AWS CodePipeline CI/CD for Spring Boot Application on ECS Fargate

## Overview

This project demonstrates a complete CI/CD pipeline for deploying a Spring Boot application to Amazon ECS Fargate using AWS CodePipeline, CodeBuild, Amazon ECR, Amazon S3, AWS IAM, and CodeDeploy.

The pipeline automatically:

* Pulls source code from GitHub
* Builds the Spring Boot application
* Creates a Docker image
* Pushes the image to Amazon ECR
* Stores build artifacts in Amazon S3
* Deploys the latest container image to Amazon ECS Fargate

## Build Process

### Source Stage

CodePipeline monitors the GitHub repository and triggers a pipeline execution whenever new code is pushed.

### Build Stage

AWS CodeBuild performs the following tasks:

1. Downloads source code.
2. Executes Maven build.
3. Creates Spring Boot JAR.
4. Builds Docker image.
5. Tags Docker image.
6. Pushes image to Amazon ECR.
7. Generates `imagedefinitions.json`.

Example:

```json
[
  {
    "name": "spring-demo-ecr",
    "imageUri": "123456789012.dkr.ecr.ap-south-1.amazonaws.com/spring-demo-ecr:build-123"
  }
]
```

---

## Artifact Storage

After a successful build:

### Amazon S3 Stores

* imagedefinitions.json
* Build artifacts generated by CodeBuild

The S3 bucket acts as the artifact store for CodePipeline.

---

## Deployment Process

AWS CodeDeploy uses the generated `imagedefinitions.json` file to:

1. Identify the target ECS container.
2. Retrieve the latest Docker image from Amazon ECR.
3. Update the ECS Task Definition.
4. Deploy the new version to ECS Fargate.

## ECS Task Definition Health Check

Configure the following health check command inside the ECS Task Definition container settings:

```text
CMD-SHELL,curl -f http://localhost:8080/actuator/health || exit 1
```

### Purpose

This command:

* Calls the Spring Boot Actuator Health Endpoint.
* Returns HTTP 200 when the application is healthy.
* Marks the container as unhealthy if the endpoint is unavailable.
* Enables ECS to automatically restart unhealthy containers.

### Spring Boot Dependency

Ensure Spring Boot Actuator is enabled:

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>
```

Expose the endpoint:

```properties
management.endpoints.web.exposure.include=health
management.endpoint.health.show-details=always
```

Test locally:

```bash
curl http://localhost:8080/actuator/health
```

Expected response:

```json
{
  "status": "UP"
}
```
